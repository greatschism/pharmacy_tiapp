Change Log - Crittercism Android Module for Titanium

08.15.2013 - v1.0.7

  • Adding NDK support.

06.14.2013 - v1.0.6

  • Fix: metadata no longer being overwritten

05.17.2013 - v1.0.4

  • Fix: now sending unhandled exceptions as crashes

05.10.2013 - v1.0.3

  • Updated the property for Crittercism App ID to "com-appcelerator-apm-id"
  • Fix: receiving uncaught JS and Java exceptions more effectively

04.05.2013 - v1.0.2

  • Updated to 3.0.10 of Crittercism library
    • See library release notes
  • Update the name of the module to "com.appcelerator.apm"
    • Updated docs to reflect above change
    • Updated sample app to reflect above change
  • Removed support for init boolean "didCrashOnLastLoad", now use API
    • Updated docs to reflect above change
    • Updated sample app to reflect above change
  • Removed support for Objects in setMetadata as this can cause unexpected behavior, will revisit
    • Note: This now accepts String objects in the value parameter
    • Updated docs and sample app to reflect above change

03.06.2013 - v1.0.1

  • Added change log to both native modules
  • Added support for "com-crittercism-ti" property in Tiapp.xml
  • Added support for optional init() parameters to reflect use of property
    • Note: adding appID to init call will override defined property.
  • Demo app currently injects the property manually
  • Documentation to reflect the changes of init parameters being optional
    • Reduced the init explanation to a single item in docs
